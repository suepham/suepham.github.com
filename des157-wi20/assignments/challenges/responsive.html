About Media Queries

Media queries allows the layout or style of the webpage to change to fit the size of the device viewing the webpage. Min-width media queries will change the layout once the minimum amount of pixels is set, so if the screen is greater than the value set, it will change. As for max-width media queries, the layout will change once the maximum amount of pixels set is reached, so any screens less than the value set will have a different layout.

Since most users are mobile users than desktop users, it is important to design for the majority of the users first. Designing mobile first helps build the content for smaller screens first and then building up from it which is easier than trying to fit a lot of content into a small screen.

Relative Units

Relative units are used in web design to scale elements based on another property. The unit "em" is based on the "font-size" property, so if the font-size was set at 12px, 2em would double the font-size. The "%" unit is based on the parent element, which could include width, height, font-size, margin, etc. Similarly, the "vw" and "vh" are based on the width and height of the viewport. It scales the element based on the size of the screen which makes it allows the style of the web page to stay consistent across different device screens.

Meta Viewport Tag

The meta viewport tag is used for responsive design because it sets the maximum amount of pixels on a screen to be the maximum amount of pixels for the webpage. This prevents the screen from zooming out and showing a larger amount of pixels than the device screen.

When I viewed the web pages with and without the meta viewport tag, the responsive page design was different. With the meta viewport tag, the web page adjusted it's layout defined in the CSS page to fit the smaller screen. However, without the meta viewport tag, the web page did not adjust the content, and the smaller screen showed a zoomed out version of the web page that is seen on a larger computer screen.
